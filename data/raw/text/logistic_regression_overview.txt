LOGISTIC REGRESSION: OVERVIEW AND MECHANICS

Logistic regression is a foundational supervised learning algorithm used for binary and
multiclass classification. Despite its name, it is a classification method rather than a
regression method in the modern ML sense. It models the probability that a given input
belongs to a particular class by applying a logistic transformation to a linear combination
of input features, producing outputs that are always bounded between zero and one and can
be interpreted as probabilities.


FROM LINEAR REGRESSION TO LOGISTIC REGRESSION

Linear regression produces outputs that are unbounded, which makes it unsuitable for
directly predicting class probabilities. Predicting a probability of 1.7 or -0.3 is
meaningless in a classification context. Logistic regression solves this by passing the
linear combination of features through the logistic function, also known as the sigmoid
function, which squashes any real-valued input to the range (0, 1).

The sigmoid function has an S-shaped curve. For very large positive inputs, the output
approaches 1, representing high confidence in the positive class. For very large negative
inputs, the output approaches 0, representing high confidence in the negative class. For
inputs near zero, the output is near 0.5, representing maximum uncertainty. This smooth
transition makes the model outputs interpretable as calibrated probabilities.

The decision boundary of logistic regression is linear in the feature space. In two
dimensions, this is a straight line. In higher dimensions, it is a hyperplane. This
linearity is both the model's main strength and its principal constraint.


LOG-ODDS AND THE LOGIT FUNCTION

An equivalent formulation of logistic regression uses the log-odds, also called the logit.
The logit is the logarithm of the ratio of the probability of the positive class to the
probability of the negative class. Logistic regression assumes that this log-odds is a
linear function of the input features. Each feature contributes an additive term to the
log-odds scaled by its learned coefficient.

Positive coefficients increase the log-odds and therefore increase the predicted
probability of the positive class as the feature value grows. Negative coefficients
decrease the log-odds. A coefficient near zero indicates that the corresponding feature
has little influence on the prediction. This interpretability of coefficients as log-odds
multipliers is one reason logistic regression is favored in domains like healthcare and
finance where interpretability is important.


MAXIMUM LIKELIHOOD ESTIMATION

Logistic regression is trained by maximizing the likelihood of the observed labels under
the model's probabilistic predictions. Equivalently, it minimizes the negative log
likelihood, also called the binary cross-entropy or log loss. For each training example,
the loss is the negative logarithm of the predicted probability assigned to the true class.

A correct high-confidence prediction incurs very little loss. An incorrect high-confidence
prediction incurs very high loss because the logarithm of a small probability is a large
negative number. This asymmetry encourages the model to avoid confident mistakes more
strongly than it is rewarded for confident correct predictions.

The loss function is convex, meaning it has a single global minimum that can be found
reliably by gradient descent. This convexity is a significant theoretical advantage over
many other models and ensures that optimization algorithms converge to the same solution
regardless of initialization.


GRADIENT DESCENT AND OPTIMIZATION

In practice, logistic regression is trained using iterative optimization algorithms such as
gradient descent, stochastic gradient descent, or second-order methods like
Limited-memory Broyden-Fletcher-Goldfarb-Shanno, commonly abbreviated as L-BFGS.

L-BFGS is the default solver in most implementations for small to medium datasets because
it uses approximate second-order curvature information to take larger, better-directed
steps than pure gradient descent. For very large datasets, stochastic gradient descent
variants that process mini-batches of data are more practical because they avoid the cost
of computing gradients over the full dataset at every step.

The choice of solver can interact with the regularization type. Some solvers only support
certain regularization penalties, so solver selection should be made jointly with
regularization choices.


MULTICLASS EXTENSION

For problems with more than two classes, logistic regression generalizes through two main
strategies. The one-versus-rest approach trains a separate binary classifier for each
class, treating that class as positive and all others as negative. The predicted class for
a new input is the one whose binary classifier produces the highest probability.

The softmax formulation, also called multinomial logistic regression, models all classes
simultaneously using a generalization of the sigmoid function that ensures predicted
probabilities for all classes sum to one. This approach is more theoretically coherent for
multi-class problems and tends to produce better-calibrated probability estimates.


FEATURE SCALING AND PREPROCESSING

Because logistic regression optimizes a loss function using gradient-based methods and
applies a linear transformation to features, the scale of input features directly affects
the optimization landscape. Features with very large values dominate the gradient
computation, while small-scale features contribute minimally. This slows convergence and
can cause numerical instability.

Standardizing features by subtracting the mean and dividing by the standard deviation
before training is almost universally recommended. This ensures that all features
contribute comparably to the gradient and allows the solver to converge in fewer
iterations.


PROBABILITY CALIBRATION

Logistic regression naturally produces probability outputs due to its sigmoid activation.
However, these probabilities may not be perfectly calibrated, meaning that among all
inputs where the model predicts 80 percent confidence, the true positive rate may not be
exactly 80 percent. Regularization and data imbalance can affect calibration.

Platt scaling and isotonic regression are post-hoc calibration methods that adjust
probability outputs after training without changing the underlying model. Well-calibrated
probabilities are essential for downstream decision-making, threshold selection, and
ranking tasks such as those encountered in ROC analysis.


ROLE IN MULTIMODAL PIPELINES

In retrieval-augmented generation systems, logistic regression can serve as a lightweight
reranking classifier that scores retrieved document candidates based on their relevance to
a query. Using features derived from embedding similarity, term overlap, and document
metadata, such a classifier can improve precision at the top of the ranking with minimal
computational overhead compared to large neural rerankers.
