REGULARIZATION IN LOGISTIC REGRESSION

Regularization is a set of techniques applied during model training to reduce overfitting
by discouraging the model from assigning excessively large coefficients to features.
Without regularization, logistic regression trained on small or noisy datasets may learn
weights that fit the training data very well but generalize poorly to unseen examples.
Understanding regularization is essential for anyone building production-grade classifiers.


WHY REGULARIZATION IS NECESSARY

In the absence of regularization, maximum likelihood optimization will push coefficients
toward arbitrarily large magnitudes when the training data is linearly separable. This
happens because perfectly separating the classes in logistic regression requires pushing
the sigmoid output toward zero or one, which requires the linear input to approach negative
or positive infinity. Coefficients that grow without bound make the model brittle, highly
sensitive to small input perturbations, and poorly calibrated.

Even when data is not perfectly separable, high-dimensional feature spaces with many
redundant or correlated features allow many different coefficient configurations to achieve
similar training loss. Regularization biases the optimization toward smaller, more
interpretable coefficients and reduces variance in the learned model.


L2 REGULARIZATION: RIDGE PENALTY

L2 regularization, also called the ridge penalty, adds a term to the loss function equal
to the sum of the squared values of all model coefficients multiplied by a regularization
strength parameter. This extra term penalizes large coefficient magnitudes, pushing all
weights toward zero without forcing any of them to be exactly zero.

The effect on the learned model is that features which do not contribute much to the loss
have their weights shrunk toward zero, while highly predictive features retain meaningful
weights. Correlated features share the predictive weight among themselves rather than one
feature monopolizing a large coefficient.

From a Bayesian perspective, L2 regularization corresponds to placing a Gaussian prior
over the coefficient values centered at zero. The regularization strength controls the
variance of this prior: stronger regularization corresponds to a tighter prior that pulls
coefficients more strongly toward zero.

In practice, L2 regularization is the default choice for logistic regression. It produces
smooth, continuous weight decay that works well across a wide range of problem types and
is supported by virtually all optimization solvers.


L1 REGULARIZATION: LASSO PENALTY

L1 regularization, also called the lasso penalty, adds a term equal to the sum of the
absolute values of all model coefficients. Unlike L2, which pushes weights toward small
values, L1 creates a penalty landscape with sharp corners at zero that actively drives
some coefficients to exactly zero.

This sparsity-inducing property makes L1 regularization valuable for feature selection.
When a model is trained with L1 regularization and strong enough penalty, many features
receive exactly zero weight, effectively removing them from the model. The remaining
non-zero weights correspond to the features the model identifies as most informative.

L1 regularization corresponds to a Laplace prior over coefficients, which has heavier
tails than a Gaussian and a sharp peak at zero. This prior structure is what produces
sparse solutions. The optimization of L1 regularized objectives is non-smooth at zero,
requiring solvers specifically designed for non-differentiable losses, such as coordinate
descent or subgradient methods.

L1 regularization is particularly useful in high-dimensional settings such as text
classification where the feature vocabulary may contain tens of thousands of terms but
only a small subset are actually predictive for a given classification task.


ELASTIC NET REGULARIZATION

Elastic net combines L1 and L2 penalties in a weighted mixture. A mixing parameter
controls the relative contribution of each penalty, allowing the practitioner to tune
between the sparsity of L1 and the stability of L2. At one extreme the elastic net
recovers pure L1, at the other pure L2, and in between it provides intermediate behavior.

Elastic net is particularly valuable when features are highly correlated. Pure L1 tends to
arbitrarily select one of a group of correlated features and zero out the rest, which can
mask genuine multi-feature effects. Elastic net allows correlated features to share weight
in an L2 manner while still promoting overall sparsity through the L1 component.


THE REGULARIZATION STRENGTH PARAMETER

The regularization strength parameter, often labeled lambda or represented as the inverse
C in many implementations, controls how heavily the penalty is applied. A large lambda
means strong regularization with small weights and high bias, leading to a simpler model
that may underfit. A small lambda means weak regularization, large weights, high variance,
and a model that may overfit.

Choosing the right regularization strength is a tuning problem best addressed through
cross-validation. A grid search over a logarithmic range of values, such as 0.0001, 0.001,
0.01, 0.1, 1, 10, 100, is a standard approach. The value that maximizes cross-validation
performance on a held-out fold is selected for the final model.

When using inverse-C notation, larger C values correspond to weaker regularization and
smaller C values to stronger regularization, which is the opposite of the lambda
convention. Being clear about which convention an implementation uses is essential to
avoid accidentally inverting the intended tuning direction.


INTERACTION BETWEEN REGULARIZATION AND FEATURE SCALING

The effect of regularization depends directly on feature scale. If features are not
standardized, regularization applies equal penalty pressure to all coefficients regardless
of whether their features are measured in large or small units. A coefficient on a feature
with small values must be large in magnitude to be predictive, and it will therefore be
penalized more heavily than a less-informative coefficient on a large-scale feature.

This imbalance distorts the regularization effect and can make the model behave
unpredictably. Standardizing all features to zero mean and unit variance before training
ensures that regularization applies uniformly across all coefficients relative to their
predictive contribution rather than their feature scale.


REGULARIZATION IN NEURAL CONTEXT

Regularization concepts from logistic regression extend directly to deep learning. L2
regularization in neural networks is equivalent to weight decay in gradient-based training.
Dropout, batch normalization, and early stopping serve analogous roles in preventing
overfitting in over-parameterized models. The core principle, that constraining model
complexity improves generalization, is universal across all supervised learning methods.
